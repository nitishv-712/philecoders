# Development Guidelines

## Code Quality Standards

### TypeScript
- `strict: true` is enforced — no implicit `any`, strict null checks apply everywhere
- Use `import type` for type-only imports: `import type { Metadata } from "next"`
- Prop types defined inline with `Readonly<{}>` for layout/component props:
  ```tsx
  export default function RootLayout({
    children,
  }: Readonly<{ children: React.ReactNode }>) {}
  ```
- No explicit `React` import needed — `react-jsx` transform handles it

### Naming Conventions
- Page/layout exports: `export default function PascalCase()`
- Font variables: `camelCase` const names, CSS variable strings use `--font-kebab-case`
- Files: `kebab-case` for directories, `camelCase.tsx` for components (App Router convention)

### Module Imports Order (observed pattern)
1. Framework imports (`next/*`, `react`)
2. Internal imports using `@/` alias
3. Style imports (`./globals.css`)

## Structural Conventions

### App Router Patterns
- Every route segment has a `page.tsx` (page) and optionally `layout.tsx` (layout)
- Root layout (`src/app/layout.tsx`) owns: `<html>`, `<body>`, font variables, global CSS
- Metadata exported as a `const metadata: Metadata` object from layout or page files:
  ```tsx
  export const metadata: Metadata = {
    title: "...",
    description: "...",
  };
  ```
- Server Components are the default — only add `"use client"` when browser APIs or hooks are needed

### Font Loading
- Use `next/font/google` for all fonts; assign CSS variables for Tailwind integration:
  ```tsx
  const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] });
  ```
- Apply font variables on `<html>` element, not `<body>`

### Image Usage
- Always use `next/image` (`Image`) instead of `<img>`
- Required props: `src`, `alt`, `width`, `height`
- Use `priority` on above-the-fold images
- Dark mode inversion via `className="dark:invert"` for SVG logos

## Tailwind CSS Patterns

### Utility Class Conventions
- Layout: `flex flex-col` as base, then `items-*` / `justify-*`
- Full-height body: `min-h-full flex flex-col` on `<body>`, `flex-1` on children to fill height
- Responsive: mobile-first with `sm:` breakpoint overrides (e.g. `sm:flex-row`, `sm:items-start`)
- Dark mode: `dark:` variant used extensively — always pair light/dark colors:
  ```
  bg-white dark:bg-black
  text-black dark:text-zinc-50
  text-zinc-600 dark:text-zinc-400
  ```
- Transitions: `transition-colors` on interactive elements (links, buttons)
- Rounded buttons: `rounded-full` with `h-12` fixed height

### Color Palette (observed)
- Backgrounds: `bg-white`, `bg-zinc-50`, `dark:bg-black`
- Text: `text-black`, `text-zinc-600`, `text-zinc-950`, `dark:text-zinc-50`, `dark:text-zinc-400`
- Borders: `border-black/[.08]`, `dark:border-white/[.145]` (alpha values via bracket notation)

## Semantic Patterns

### External Links
Always include `target="_blank"` with `rel="noopener noreferrer"` on external anchors:
```tsx
<a href="https://..." target="_blank" rel="noopener noreferrer">
```

### CSS Custom Properties
Tailwind v4 uses CSS variables for theming; `--font-*` variables bridge `next/font` with Tailwind's `font-sans` / `font-mono` utilities.

### Config Files
- `next.config.ts` uses TypeScript with explicit `NextConfig` type annotation
- Keep config minimal; add options only when needed
- Never edit `next-env.d.ts` — it is auto-generated by Next.js

- Next.js 16 has breaking changes — consult `node_modules/next/dist/docs/` before using any API not confirmed in this codebase (per AGENTS.md)
